# aula_max_dia-15-04

Considerando os requisitos funcionais do Sistema FitPass Gym Management, elabore o diagrama de classes do domínio. Salvar os docs em um repositório do GitHub e postar o link. <br>

RF01 — Cadastro de **Alunos** <br>
O sistema deve permitir cadastrar novos **alunos** contendo dados pessoais, contato, endereço e plano contratado. <br> 

RF02 — Gerenciamento de **Planos** <br>
O sistema deve permitir criar, editar, ativar e desativar tipos de **planos** (mensal, trimestral, anual, musculação, funcional etc.). <br>

RF03 — Controle de **Pagamentos** <br>
O sistema deve registrar pagamentos realizados na recepção (dinheiro, cartão ou PIX) e gerar boletos/cartões para pagamentos online. <br>

RF04 — Regularidade do Aluno <br>
O sistema deve verificar automaticamente se o aluno está com mensalidade em dia. <br>

RF05 — Controle de **Acesso** <br>
O sistema deve se integrar à catraca para validar a entrada do aluno por RFID. <br>

RF06 — **Agendamento** de **Aulas** <br>
O aluno deve poder visualizar horários e reservar vagas nas aulas disponíveis. <br>

RF07 — Lista de **Presença** <br>
**Instrutores** devem registrar presença dos alunos nas aulas. <br>

RF08 — **Avaliação Física** <br>
Instrutores devem registrar avaliações físicas (peso, IMC, percentual de gordura etc.) e anexar arquivos se necessário. <br>

RF09 — Relatórios Gerenciais <br>
O **gerente** deve poder emitir relatórios como: <br>

inadimplência <br>
alunos ativos <br>
histórico de acessos <br>
ocupação das aulas <br>

RF10 — **Notificações** <br>
O sistema deve enviar notificações ao aluno sobre: <br>

vencimento de mensalidade <br> 
confirmação de agendamento <br>
liberação de nova avaliação física <br>
